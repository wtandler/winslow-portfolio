## The Idea

Most intake processes are a queue of requests and a human reading every one to decide what it is, how urgent it is, and where it goes. That triage is mechanical right up until the moment it needs judgment—which makes it a near-perfect fit for an agent that does the first pass and hands humans only the calls that actually need a person.

## What the Prototype Does

A working front-end for that pattern:

* **Submit** — a request comes in through a simple form.
* **Queue** — requests land in a triage queue, each tagged with an AI **confidence** and **status** so the obvious ones are obvious.
* **Review** — a human opens anything the agent is unsure about, with the context already assembled.
* **Analysis** — a view over what's flowing through the system.
* **Admin** — configuration for the business units the intake routes to.

## Status

This one is earlier than the others on this page—a functional prototype built to prove out the submit → triage → review → analysis loop and the AI-confidence UX, not yet a shipped product. The interesting question it's testing: how much of an intake desk can an agent own before a human needs to step in, and how do you make that handoff legible?
